# 有道自习室 STUDIO · 设计系统规范 v2

> 来源：Claude Design 交付的《有道自习室 STUDIO v2》完整高保真原型（9 页 + 亮暗双模 + 三档响应式）。
> 本文档从原型 bundle 中精确提取全部设计 token，作为 tide-work 前端重构的施工蓝图。
> 提取自 `有道自习室 STUDIO v2 (standalone).html`（2026-07-03）。

---

## 0. 核心设计主张（重构时的最高准则）

- **产品命名**：「有道自习室 · STUDIO」（弃用"潮汐"，"自习室"隐喻更易读）。副标 `STUDIO · YOUDAO`。
- **叙事**：把学习包装成"点亮自习室"——每天来学 = 点亮。streak / 日历热力 / "再学 2 天解锁徽章"围绕此隐喻。
- **红色纪律（最关键）**：品牌红 `#FC011A` 全局占比 **~7%**，只承载**单一语义 = "专注信号 / 当前焦点 / 主行动"**。
  - 红只出现在：当前进度（进度条/百分比）、当前学习断点（大纲"在学"行整行高亮）、主推 CTA、主推订阅卡、streak 火苗、共创第一名、日历热力深浅。
  - 中性色是**冷灰蓝（明确"非米棕"）**，大面积是灰白/深灰蓝。
  - `NEW` 徽章用**米黄** `#FFF3D0`（刻意避开红，不与"专注信号"抢注意力）。
- **气质**：安静、有空间感、软深度（用双层柔投影而非硬边框做层次）。
- **布局范式**：桌面 App 工具式——**左侧固定侧边栏导航 + 右侧主内容区**（像 Notion/Linear/得到），不是营销站的顶部导航。
- **禁忌**（用户明确排除）：无水墨、无具象流水、无米棕"Claude"色系、不堆砌花哨多彩。

## 1. 颜色 Token

### 1.1 亮色模式（默认）

```css
:root {
  /* 背景层（冷灰蓝，从外到内递进） */
  --bg:            #E7EAF0;  /* 最外层页面底 */
  --bg2:           #EDEFF3;  /* 次级底 */
  --surface:       #FFFFFF;  /* 卡片/表面白 */
  --surface2:      #F4F6F9;  /* 次级表面 */
  --surface-inset: #EAEDF2;  /* 内凹表面（输入框等） */
  --border:        #E2E6EC;  /* 主描边 */
  --border2:       #D3D9E2;  /* 次级描边 */

  /* 文字（冷灰蓝墨色，四阶） */
  --ink:  #232935;  /* 正文/标题 近黑 */
  --ink2: #5B6474;  /* 次级文字 */
  --ink3: #8790A0;  /* 三级文字（⚠️暗色下对比度偏低，见 §7） */
  --ink4: #AEB6C2;  /* 最弱/占位 */

  /* 品牌红（唯一强调色，~7% 占比） */
  --red:             #FC011A;  /* 品牌红（锁定，不可替换） */
  --red-soft:        #FFF1F2;  /* 红调浅底 */
  --red-soft-border: #FFD9DD;  /* 红调浅描边 */
  --red-ink:         #C8000F;  /* 红字（需对比度时用深一档） */
  --red-400:         #FF5462;  /* 红渐变亮端 */

  /* 视频/深色区（亮色模式里的深色块，如视频占位） */
  --video-bg: #232935;

  /* NEW 徽章（米黄，刻意非红） */
  --new-bg:  #FFF3D0;
  --new-ink: #8A6A00;

  /* 辅助蓝（极少量，如某些图标） */
  --accent-blue: #2E6BE6;

  /* 遮罩 */
  --scrim: rgba(20,23,28,.55);
}
```

### 1.2 暗色模式（`[data-theme="dark"]`）

```css
[data-theme="dark"] {
  --bg:            #0E1116;  /* 冷调深，非纯黑 */
  --bg2:           #12151B;
  --surface:       #191D25;
  --surface2:      #20252F;
  --surface-inset: #20252F;
  --border:        #2B3350;  /* 冷蓝调描边 */
  --border2:       #39415E;
  --ink:  #EDEFF3;  /* 暗色下正文转浅 */
  --ink2: #AEB6C2;
  --ink3: #8790A0;  /* ⚠️ 对比度偏低，暗色下建议提到 ink2 或加大 */
  --ink4: #5B6474;
  /* 红/soft/new 系保持不变（红在暗色下更聚焦） */
  --video-bg: #12151B;
}
```

> 提取到的深色描边 `#2B3350 / #39415E` 是冷蓝调，与整体"冷灰蓝"主张一致。

## 2. 字体系统

```css
/* UI 主字体：中英双字体分工 */
--font-ui: 'Plus Jakarta Sans', 'Noto Sans SC', system-ui, sans-serif;
/* 数据/数字：等宽，用于价格、时长、票数、streak 天数 */
--font-mono: 'IBM Plex Mono', ui-monospace, monospace;
```

- **中文**：Noto Sans SC（思源黑体，开源网页字体）
- **英文/UI**：Plus Jakarta Sans（开源，几何人文感）
- **数据**：IBM Plex Mono（等宽，专业感）——价格 `¥298`、时长 `15:00`、票数 `1,284`、streak `28` 都走等宽

### 字号阶梯

| 名称 | px | 用途 |
|---|---|---|
| Display | 40 / 32 | Hero 大标题、订阅价格 |
| Heading | 24 / 22 | 页面/区块标题 |
| Sub | 18 | 卡片标题 |
| Body | 15 | 正文 |
| Small | 13 | 次级信息 |
| Data | 12 | 等宽数据标签 |
| Caption | 11 | 最小标注/overline |

## 3. 间距 / 圆角 / 阴影

### 圆角（从提取频次看，主用这几档）
```css
--radius-sm:   8px;    /* 小标签/输入 */
--radius-md:   12px;   /* 卡片内元素 */
--radius-lg:   16px;   /* 卡片 */
--radius-xl:   18px;   /* 大卡片/视频区 */
--radius-pill: 999px;  /* 胶囊/徽章/按钮（高频，24次） */
```

### 间距（gap 常用值）
```
4px / 8px / 9px / 10px / 12px / 16px  （8px 基准节奏）
```

### 阴影（"软深度"——双层柔投影，非硬边框）
```css
/* 卡片默认 */
--shadow-card: 0 2px 6px -2px rgba(35,41,53,.12);
/* 抬起/hover（双层：投影 + 内高光） */
--shadow-lift: 0 10px 26px -12px rgba(35,41,53,.28), 0 1px 0 rgba(255,255,255,.7) inset;
/* 阴影基色 rgba(35,41,53,...) = --ink 的 RGB，柔和不发灰 */
```

## 4. 动效系统（提取自 @keyframes，安静 + 物理感）

```css
/* 时长 180–500ms，spring/ease 缓动 */

/* 进场：内容上浮淡入 */
@keyframes riseIn { from{opacity:0;transform:translateY(10px)} to{opacity:1;transform:translateY(0)} }
/* 面板：侧向滑入（如笔记面板） */
@keyframes slideIn { from{opacity:0;transform:translateX(14px)} to{opacity:1;transform:translateX(0)} }
/* 录制脉冲：记笔记时的红点呼吸 */
@keyframes recPulse { 0%,100%{opacity:1;box-shadow:0 0 0 0 rgba(252,1,26,.5)} 50%{opacity:.75;box-shadow:0 0 0 5px rgba(252,1,26,0)} }
/* 截帧闪光：截取此帧时的白闪反馈 */
@keyframes flashCap { 0%{opacity:0} 12%{opacity:.85} 100%{opacity:0} }
/* 悬浮：图标/元素轻微上下浮动 */
@keyframes floatY { 0%,100%{transform:translateY(0)} 50%{transform:translateY(-6px)} }
```

**过渡曲线**：
```css
--ease-out: cubic-bezier(.2,.7,.2,1);   /* 主用，自然减速 */
--t-fast: .12s;  --t-base: .18s;  --t-slow: .2s;  --t-progress: .4s;
```

**动效命名哲学**：`进场 riseIn · 面板 slideIn · 卡片 hover-lift(translateY(-3)+柔投影) · 录制 recPulse · 截帧 flash`。安静、有物理感、时长克制。

## 5. 组件规范

### 按钮体系
- **主按钮**：浅色实心 + `▸` 图标（暗色页里的浅色 CTA，如"进入工作台"）
- **强调红按钮**：红色实心（仅主推 CTA，如订阅"立即开通"）
- **次按钮**：描边（如"浏览课程"）
- **文字按钮**：无边框
- **状态按钮**：`记录中`（红点 + 描边胶囊，recPulse 呼吸）

### 卡片
- 圆角 16-18px + `--shadow-card`，hover 时 `translateY(-3px)` + `--shadow-lift`
- 课程赛道卡：图标 + 标题 + "+N 节新课上新"，当前赛道深色高亮

### 徽章/标签
- 分类标签：红色胶囊（`英语口语`）
- `NEW`：米黄胶囊（非红）
- streak：`🔥 28天`（火苗 + 等宽数字）

### 进度
- 进度条：红色填充（当前进度语义）
- 百分比：等宽大数字 + 红色

## 7. 落地注意事项

1. **无障碍**：暗色模式下 `--ink3 (#8790A0)` 对比度偏低。tide-work 有银发用户群，落地时应把暗色下三级文字提到 `--ink2` 或加大，满足 WCAG AA。
2. **布局是换血不是改色**：现 tide-work 是"暖白 + 顶部导航"营销站，这套是"冷灰蓝 + 左侧导航"App 工具布局。重构 = 整体架构调整，需分阶段。
3. **字体需引入**：Plus Jakarta Sans / Noto Sans SC / IBM Plex Mono（都是开源可自托管，避免 CDN 依赖 —— tide-work 现用 Geist，需替换）。
4. **红色纪律要写进 lint/review**：全局红占比 ~7% 是硬约束，重构时每个页面都要自查红色是否只用在"专注信号"语义上。
5. **动效用 CSS/transform 实现**（riseIn/slideIn/recPulse/flashCap），GPU 友好，与之前的动效性能优化方向一致。
