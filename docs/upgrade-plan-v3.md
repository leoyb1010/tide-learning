# 有道自习室 STUDIO · v3.0 升级优化计划

> 基线：v2.3 + 三轮视觉升级（commit 864234b，Web + iOS 同仓）。
> 输入：用户 9 条实测反馈 + 三路代码侦察（造课链路 / 性能根因 / 设置·订阅·布局·分享现状）+ 自主产品体检。

## §1 造课革命（F2 · 本次最重的一块）

用户原话：「最重要的就是造课出来的样式，如何分章节，如何让用户愿意用我们平台来持续学习」。拆成三个子战役：

### 1.1 后台生产 + 断点续造（架构修复）

**目标**：点「开始造课」后，关页面、切页面、刷新，生成照常进行；随时回来看到实时进度。

方案（双轨，都要做）：

**轨道 A：服务端接管生成（真后台）**
- `POST /api/ai/generate-course` 落库大纲后，用 Next.js 15 `after()` 在响应返回后继续在服务端跑「逐节生成循环」（复用现有 generate-lesson 内部逻辑，抽成 `generateLessonCore(lessonId)` 供 route 和后台循环共用）。
- `GenerationJob` 表从「记账」升级为「任务队列语义」：course 级 job 记录 `total/done/failed/currentLessonId`，每节完成后更新。幂等锁：genStatus=generating 且 job running 时拒绝重复触发。
- 服务端进程重启的兜底：任何读到 `genStatus=generating` 且 job 超过 10 分钟无心跳的课，标记 `stalled`，UI 提供「继续生成」按钮触发续跑（从第一个 blocksJson=null 的节开始）。

**轨道 B：全局进度可见 + 恢复 UI**
- 新增 `GET /api/courses/[id]/gen-progress`：返回 `{total, done, failed, current, phase}`。
- `/create` 页进入时检测「我有 generating 中的课」，顶部显示**生产中横幅**（课名 + 进度环 + 点击回到剧场）。
- 过程剧场组件改造：支持从任意进度**水合恢复**（拉大纲 + 各节状态，续播动效），轮询 gen-progress（3s，页面可见时才轮询）。
- `/me/courses` 生成中的课卡：进度环 + 「查看进度」直达剧场；失败节自动列出可重试。
- TopNav 增加轻量「生产中」指示（有 generating 课时出现小进度点，点击直达）。

### 1.2 课件块协议升级（内容形态）

现有 5 种块太薄。扩展为 **12 种块**，让每节课有「杂志感 + 交互感」：

| 新块类型 | 用途 | 渲染形态 |
|---|---|---|
| `objectives` | 节首学习目标 | 章节页头卡：本节你将学会 N 件事，逐条 stagger 亮起 |
| `scene` | 场景引入/钩子 | 深色渐变卡（--video-grad）+ 大字引言，把「为什么学」讲成故事 |
| `dialog` | 对话示例（口语课刚需） | 聊天气泡左右分列，播放式逐条浮现，可跟读标记 |
| `steps` | 步骤教程 | 竖向步骤条，序号圆点 + 连接线，当前步高亮 |
| `compare` | 对比表 | 双栏对比卡（适合「常见误区 vs 正确做法」） |
| `example` | 例子/案例 | 缩进引用卡 + 「例」角标 |
| `flashcard` | 内联翻转卡 | 点击 3D 翻面（复用复习室翻牌），一键存入复习队列 |
| `summary` | 节尾小结 | 与 objectives 呼应的收束卡 + 「下一节预告」钩子 |
| 保留 | concept/code/quiz/keypoint/callout | 渲染同步升级（见 1.3） |

配套：
- `blocks.ts` 白名单 + 校验扩展；未知类型继续前向兼容。
- **生成 prompt 重写**：给 LLM 一个「节结构模板」：`scene(钩子) → objectives → concept/dialog/steps 交替 → quiz(节中检查) → keypoint → summary(含下节预告)`。要求每节至少 1 个交互块（quiz/flashcard），口语类课强制 dialog 块。这是「样式好看」的根，内容结构先立起来。
- 旧课兼容：已有 5 种块的课照常渲染，不迁移。

### 1.3 课件渲染升级（视觉与动效）

学习页（Player 的 ai_block 路径 + BlockRenderer）按「沉浸阅读刊物」标准重做：

- **章节仪式感**：节首页头 = 章节号大字（mono）+ 标题 + 预计时长 + objectives 卡；节间有「你已完成 x/y」的呼吸分隔。
- **滚动叙事**：每个块 IntersectionObserver 进场（rise + 交错），quiz 答对时 num-pop + confetti 微粒（克制），keypoint 存卡成功有飞入复习室的轨迹动效。
- **阅读进度**：右侧细水位线（复用 WaveProgress 语义），读完自动标记完成 + Haptics（iOS）。
- **排版**：正文限宽 68ch、行高 1.8、中文字间距优化；code 块暗色主题统一；块间距节奏 8pt 网格。
- 全部动效 reduce-motion 降级；IO 懒挂载保证长课件性能。
- **iOS 同步**：BlockCardView 增加同批新块类型渲染（dialog 气泡/steps/flashcard 翻面 + 触觉）。

## §2 全局分享体系（F3 · 增长回路）

原则：**每个「值得炫耀的时刻」都要能一键变成一张好看的卡片**。

### 2.1 分享图生成服务（服务端，一次建好多处复用）
- 技术：`ImageResponse`（next/og，satori），零额外依赖，SVG 排版输出 PNG。
- 统一路由：`GET /api/share-card/[kind]?...`，kind 覆盖：
  1. `student-card` 学生证卡（昵称/学号/Lv/连续天数/格言/二维码 → 指向 /u/[id]）
  2. `week-report` 学习周报（本周分钟/笔记数/完课/连击最高，柱图曲线）
  3. `course-done` 完课证书卡（课名/用时/日期/编号）
  4. `note-quote` 笔记摘录卡（笔记内容排版 + 课程来源 + 品牌角标）
  5. `streak` 连续 N 天里程碑卡（7/30/100 天触发）
  6. `exam-result` 模拟考成绩单卡
- 视觉：全部走 STUDIO 语言（冷灰蓝 + 红点睛 + mono 数字 + 潮汐水纹底），1080×1440（竖版，适配朋友圈/小红书）+ 1200×630（横版 OG）双规格。

### 2.2 分享面板组件（前端统一入口）
- `<SharePanel />`：预览分享图 + 三个动作（下载图片 / 复制链接 / 系统分享 navigator.share）。
- 挂载点：学生证（/me）、完课时刻（学习页结算）、复习结算页（连击/清空）、笔记详情、模拟考成绩单、课程详情（分享这门课）。
- 课程详情页动态 OG：`opengraph-image.tsx` per course（标题 + 赛道渐变 + 学习人数），微信/X 粘贴链接即出卡。

### 2.3 iOS 分享
- `ShareLink` + 服务端分享图 URL（复用同一套 /api/share-card），学生证/成绩单/周报三处先接。

## §3 性能根治（F1 · 快赢优先做）

### 3.1 数据层（prod 也慢的真问题）
- `/api/notes` 加 cursor 分页（take 30 + nextCursor），前端滚动加载；笔记搜索/筛选走服务端参数。
- notes 页首屏改 **Server Component 直查**（首屏 30 条 SSR），交互区（视图切换/筛选/AI 整理）拆成客户端岛。骨架与真实布局对齐。
- 列表虚拟化：> 60 条时启用（首选 CSS `content-visibility: auto` 零依赖方案，不够再上 react-window）。
- `/api/desk` 等聚合 route 审一遍 select 收窄（只取渲染需要的字段）。

### 3.2 交互层（点两次/无反馈）
- AI 整理菜单：busy 加 15s 超时兜底复位；请求中菜单项显示 spinner 而非静默 disabled。
- 新建笔记本/发帖/评论等所有提交按钮：统一 `useSubmitGuard`（提交中 disabled + loading 态），杜绝双击双发。
- TopNav 下拉：监听器改挂载一次的事件委托；菜单项用 `onPointerDown` 提前响应。
- 全站按钮点击 100ms 内必须有视觉反馈（studio-press 已有，补漏网的裸 button）。

### 3.3 运行模式（要向用户说清）
- dev 模式（next dev）按需编译，每个首访路由都要现编译，**慢是本质**。给本地测试脚本 `npm run preview`（build + start，端口 3100），体验即为真实线上速度。验收以 prod 模式计。
- next.config 补 `images` 优化域、`compress`；重组件（ExamRunner/CreateStudio 剧场）dynamic import 降首包。

**验收**：prod 模式下，笔记馆 500 条数据首屏 < 1s，路由切换 < 300ms，无任何「点两次」复现。

### 7.1 修当下的破绽
- 「最受欢迎」角标只留一个（放年卡），修复角标被卡片 overflow 裁切。
- 三档权益不再复制粘贴：按档位递进呈现差异。

### 7.2 定价心理学重构
- **锚定**：年卡为推荐主卡（视觉最大 + 红 CTA + cta-glow + 「省 ¥xxx」标签），月卡做入门锚点，季卡中间过渡。

- **首期优惠**：连续包月首期价视觉强化 + 「随时取消」保障并列。

### 7.3 与产品体系真联动（关键差异化）
- **订阅送积分**：月卡月赠 300 / 季卡月赠 500 / 年卡月赠 800 积分（积分用于 AI 造课/AI 整理，直连 CreditLedger 新 type=`subscription_grant`，随订阅周期发放）。订阅卡上明确写「每月赠 xxx 积分，可造 x 门课」。

- **档位徽章**：订阅者学生证加档位标识（年卡金边之类，克制）。

### 7.4 说服力组件
- 顶部价值区：不是「全站会员」四个字，是「一次订阅，AI 帮你把想学的都造成课」+ 三个数字（课程数/学习人数/周更新数）。
- FAQ 手风琴（退订/发票/多端/积分规则）。
- 底部「随时取消，笔记永远是你的」信任条 + 集市学员心得引用 2 条。
- iOS SubscriptionView 同步这套结构（StoreKit 价格本地化）。

## §8 自主升级建议（产品 / 可视化 / UI / 动效）

除用户 9 条外，我对产品的体检建议，按 ROI 排序：

### 8.1 产品能力（留存回路）
1. **学习周报**（强推）：每周一自动生成（本周分钟/完课/笔记/连击/对比上周），书桌横幅 + 可分享（接 §2）。留存产品的标配。
2. **首次引导 Onboarding**：新用户 3 步引导（选赛道兴趣 → 说一个想学的 → 看着 AI 造出第一门课），把「啊哈时刻」压缩到 90 秒内。现在新用户进来是空书桌。
3. **Streak 保护**：连续天数是最强留存钩子，但断一天归零太残酷。加「补签卡」（积分兑换，每月限 1）。
4. **课程社会证明**：课程卡/详情加「本周 N 人学过」「完课率 x%」，集市课加学员评分。
5. **AI 伴侣主动化**：伴侣从「被动问答」升级为「场景触发」：学完一节主动出 3 题、复习连错主动讲解、周报里点评学习节奏。

### 8.2 可视化升级
6. **学习热力图年视图**：/me 加 GitHub 风格 365 天热力格（数据已有 calendar），是最有「晒」价值的图。
