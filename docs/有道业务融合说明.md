# 有道学习会员 · 业务融合说明（v0.4）

> 本文记录《学习会员产品介绍 1.0》(有道) 与本平台的融合落地。潮汐学习是有道学习会员模式的**平台化/通用化版本**,两者高度同构,本次将有道真实业务的关键能力全量落地。

## 一、融合判断

| 维度 | 有道学习会员 | 潮汐(已建) | 处理 |
|---|---|---|---|
| 订阅制/降门槛/持续更新/全龄/多内容线/免费试学 | ✅ | ✅ | **同构,直接对齐** |
| 一手流量引擎(词典私域+投流) | ✅ | ❌ | **补:渠道埋点 + 建联漏斗** |
| 0转正 + 电联建联 | ✅ | ❌ | **补:预约试听 + 建联队列** |
| 直播小班课 | ✅ | ❌ | **补:live 章节类型** |
| 需求共创投票 | ❌(靠投流数据) | ✅ | **潮汐反哺:投票+数据双输入** |
| 弱主讲依赖 | ❌(主讲要50%分成) | ✅ | **潮汐反哺:contributor+分成核算** |
| 分赛道自由组合订阅(2026H2) | 规划中 | ❌ | **补:scoped entitlement** |

## 二、三层落地(全部完成)

### 🟢 Tier 1 — 对齐真实业务
- **定价**:全站连续包月 ¥99(首月¥19.9)/季卡¥249/年卡¥499 + **新增季卡**;单赛道月卡 ¥19.9–29.9(`prisma/seed.ts`)
- **内容赛道**:AI/生活 → 融合有道英语板块。`src/lib/tracks.ts` 定义 5 条赛道:口语实战 / 听说读写全能(躺学单词) / 银发口语 / AI 技能 / 生活实用;未来赛道(职教/考证/留学/亲子)预留
- **渠道来源**:`analytics_events` + `Lead.source` 三渠道维度(端内私域/端外投放/私域),后台看板按渠道拆漏斗

### 🟡 Tier 2 — 补短板
- **预约试听 + 电联建联漏斗**(有道最核心转化):`Lead` 表 + `POST /api/leads` 留资 + `/admin/leads` 建联队列(support 角色跟进:new→电联中→已预约→试听中→转化/流失)
- **直播小班课**:`Lesson.contentType="live"` + `liveStartAt/liveSeatLimit`,播放器 `LiveBanner` 展示开播时间与预约
- **渠道×人群看板**:`/admin` 新增渠道漏斗(曝光→留资→转化)+ 分赛道订阅分布

### 🔴 Tier 3 — 架构演进(对应有道 2026H2)
- **分赛道自由组合订阅**:`Plan.scope`(all/单赛道)+ `Subscription.scope` 快照;`resolveEntitlement` 归约 `accessibleTracks`("all"|赛道列表),支持**多个单赛道订阅叠加**;`canAccessTrack/canAccessLesson` 按赛道鉴权
- **弱主讲依赖**:`Course.contributorName/revenueSharePct` 字段 + 共创投票接入选题排期(`ContentCalendar.demandId`),投票 + 投流数据双输入降低单一主讲议价权

## 五、两个取舍(保留在产品判断里)

1. **投票共创 vs 投流数据**:不二选一。投票做参与感与续订理由,投流数据做真实选题依据,两者喂同一个 `ContentCalendar`。
2. **直播是双刃**:直播转化好但重人力(正是有道想摆脱的成本)。落地为**录播为主、直播为钩子**,而非直播主体。
