## 〇、愿景终态:蓝图完成时,产品能做到的 6 件事

1. **一句话 → 一门精品课,全自动**:大纲 → 逐节内容 → 课级设计系统 → 强模型 bespoke 精修 → 插图/图表 → 封面 → 分享卡/视频/音频,全链自动;任何一环失败自动降级自愈,**永不空课、永不难看**。
2. **每门课都好看,且看得出不同**:确定性底座本身达到「高级」水准(封面系统、构图节奏、插图、图表);强模型可用时再上一档;盲测能凭首屏分辨模板与赛道。
3. **质量自主化,不靠人盯**:生成 → 结构机检 → 视觉机检 → 不达标自动重生成/升模 → 抽样人审墙。低质课到不了用户面前。
4. **学习闭环全接通**:课件内答题/翻卡 → 错题本/复习卡(SRS)/进度/streak/成就/周报,一条数据不漏;复习室、模拟考、笔记馆共用同一套学习数据。
5. **分享即增长**:每门课自动产出分享卡、OG 图、免登录预览页、课程预告视频;二维码落地页精美;集市交易 + 创作者分成形成生态环。
6. **模型即插即用**:模型注册表多模态化(text/tts/image/video + 延迟/上下文元数据),接新模型 = 加一条注册表记录;强弱模型各有对应产出档,没有任何一档「交了钱拿保底」。

## 一、审查结论(为什么蓝图长这样)

> 审查方法:① 逐行核对 7·12《课件生成能力评估》6 条建议落地状态;② 精读 `src/lib/ai/` 全部 12 文件与 `course-gen.ts`;③ 两路并行代码调研(学习端渲染+分享闭环 / 造课全链路+模型层);④ 浏览器实测 6 节确定性课件(4 套 art);⑤ **公网 DeepSeek 按管线逐字同款 prompt 真实生成 1 节 bespoke 并过仓库 QC 门**(一手实验)。

**四个关键事实:**

1. **上一轮 6 条修复建议 0/6 落地**,现库 46/46 节课件全部 `deterministic`、0 节 LLM 精修——天花板路径现网利用率 **0%**。一手实验:同款 prompt + deepseek-chat,**32.3s 一次通过全部 QC 门**。**瓶颈不在模型,在管线三道闸门**(白名单耦合、45s+0 重试、QC 整节拒收)。
2. 确定性底座架构精良(14 块 × 12 art × 9 mode × 8 模板 × variance/motif × 双模式运行时),但有 **5 个系统性观感硬伤**(见发现表 P2),全部零模型依赖可修——这是「保底不够高级」的真实来源。
3. **平台闭环三处断电**:课件答题不回传(学习数据白流);`recordActivity` 全仓零调用(streak/热力图/成就/3 张分享卡对真实用户全空);集市/主页无 OG 图(分享二维码落到「白板」)。
4. 工程债:无内容审核层、`after()` 非持久无真队列、无版本化、无流式、免费用户 0 造课权(冷启动死锁)。

## 二、终态架构:「不对称三层 + 自愈闭环 + 多模态注册表」

```
┌─ 内容真值层  blocksJson(14块协议)      任何模型都能产合法块 → 下限保底
├─ 表现自由层  bespoke HTML / 插图 / 视频  强模型自由发挥 → 上限不封顶
└─ 协议壳层    RUNTIME · ct-* · CSP · 回传  平台注入,与模型无关 → 永远一致
        ↑ 质量自愈闭环:机检 → 视觉检 → 重生成/升模 → 抽样人审
        ↑ 模型注册表:key/tier/costWeight + latencyTier/maxOutput/modality
```

**地板由确定性系统保证,天花板由模型自由发挥,中间用协议壳与自愈 QC 焊接。**弱模型 → 确定性渲染也高级;强模型 → 拆闸后自然兑现;新模型/视频模型 → 注册表加一行,表现层多一种资产,架构不动。这就是「模型能力永不设限」的工程落法。
