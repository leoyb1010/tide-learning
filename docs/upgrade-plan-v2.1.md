## 总纲：一个判断统领全部

当前产品的根病灶：**信息架构还是"平台视角"（我有什么课卖给你），不是"自习室视角"（你今天来学什么）。**

自习室的本质体验 = 「我的位置 + 我的资料 + 我的进度 + 一个随叫随到的助教 + 不被打扰的专注」。
①首页 ②导航 ③造课体现 ④导入价值 ⑤笔记 ⑦社区 ⑨专注 是体验层；⑥性能 ⑧⑩设计与动效 是质量层。

## ① 首页：从「卖课橱窗」改成「我的自习桌」

**问题**：登录后首页仍是营销页结构（Hero 口号 + 课程赛道 + 订阅推销）——是"平台想让你看什么"，不是"你的自习室"。

**方案：双态首页**

**A. 未登录 = 门厅**：保留营销结构（Hero/赛道/订阅），讲清价值引导注册。

**B. 登录后 = 自习桌 dashboard（核心改造）**，自上而下：
1. **问候 + 今日状态**：「下午好，安琪 · 已连续 14 天 · 今天还没点亮」
2. **中央大输入框「今天想学点什么？」**（自习室灵魂交互，三合一）：
   - 输入问题 → AI 伴侣直接答
   - 输入想学主题 → 引导 AI 造课
   - 输入关键词 → 搜课程/笔记
3. **继续学习卡**（断点续学）
4. **我的书桌**（横排）：我的课（AI 生成/导入）+ 最近笔记 3 条 + 待复习卡数
5. **AI 今日建议**：「继续《30天口语》第9课 · 复习 5 张卡 · 你上周在'连读'卡了 2 次，建议重看 L08」
6. **自习室氛围条**：「此刻 1,284 人在自习」+ 进入专注模式（与 ⑨ 联动）
7. 课程推荐降权为底部一小节「书架上新」

**涉及**：`src/app/page.tsx` 登录态分支、新组件 `StudyDesk.tsx`、复用 gamification/progress/notes 查询。

## ③ AI 造课能力显性化：「备课剧场」

**问题**：生成过程是黑盒，后端的大纲/逐节/测验能力用户感受不到。

**方案**：生成开始后进入专属过程页，实时演出 AI 备课：

```
🤖 AI 正在为你备课《Python 装饰器入门》
  ✅ 理解需求：初学者 · Python · 装饰器
  ✅ 搭建大纲：7 节课，由浅入深（大纲逐条浮现）
  ⏳ 正在写第 3/7 节「wrapper 与参数透传」
      └ 概念讲解 ✓ · 示例代码 ✓ · 随堂测验 ⏳
  ⬜ 第 4-7 节排队中
```

- 每节完成即可点进去学（等待变期待）
- 完成页「这门课包含」：N 节 · N 测验 · N 要点卡 · AI 伴侣已就位
- **零新后端**：现有 generate-course/generate-lesson 已分步，前端剧场化即可

**涉及**：`CreateStudio.tsx` 重构 + 生成过程视图。

## ④ 导入外部资料的竞争力：「资料升维」

**问题（最尖锐）**：用户为什么要把收集的资料导入这里学？导入若只是"换个地方看"，毫无竞争力。

**答案：导入不是搬运，是升维。原始资料 → 一门活的课。** 用户得到 6 样原来没有的：

| 原始资料只是… | 导入后变成… |
|---|---|
| 要自己啃的文本 | **结构化章节**（循序渐进） |
| 看完就忘 | **自动随堂测验** |
| 有疑问没人问 | **AI 伴侣基于原文答疑（带引用，不虚构）** |
| 划线散落 | **锚定在内容上的笔记**，沉淀个人知识库 |
| 从不复习 | **要点自动成复习卡** + 遗忘曲线提醒 |
| 学没学无感 | **进度/streak/热力日历** |

**竞争力表述**：笔记工具能存资料、网课平台能卖课，**没有产品把"你自己的资料"变成"带测验、带助教、带复习系统的课"**。与 ⑤ 知识库互相强化。

**产品化落点**：
1. 导入页文案重写：不说"导入资料"，说 **「把任何资料变成一门你的课」**，排出 6 项收益
2. 导入完成页 = **「升维报告」**：你的 8,000 字资料 → 6 章 · 12 测验 · 9 要点卡 · 伴侣已读完全文
3. 首页输入框支持直接贴文本触发导入
4. P1 扩源：URL 抓取（防 SSRF）、PDF；P2 视频转写

## ⑥ 性能：真相与对策（已实测诊断）

**实测**：已编译后各页 SSR 仅 **10-50ms**——服务端不慢。1-3 秒卡顿来自三个叠加：
1. **dev 模式按需编译（主因）**：每页首次访问现场编译 1-3s。**生产构建后此项消失**——上线反而快一个数量级
2. **dev 禁用 Link 预取**：生产下 `<Link>` 自动 prefetch 视口内页面（点击即达）
3. **入场动画 0.5-0.7s**：页面已加载但视觉上"还在等"

**对策（按见效排序）**：
- 先本地跑生产模式验证：`npm run build && PORT=3100 npm start`
- 动画减负：入场 0.5s→0.25s，仅首屏第一区块用，列表不做逐项延迟
- 每路由补 `loading.tsx` 骨架屏（点击瞬间反馈，感知延迟≈0）
- **resolveEntitlement 去写化**：现在每次页面加载执行一次 `updateMany` 写库；改为读时内存判断过期 + 节流写（生产 Postgres 下的隐患）
- React `cache()` 去重同请求内的 getCurrentUser/resolveEntitlement 重复调用

## ⑩ UI / 动效 / 布局系统升级（专章）

> 现状判断：STUDIO v2 打好了 token 地基，但动效是「通用库存动效」没有产品记忆点；布局有骨架但卡片/间距执行有漂移；关键时刻（点亮/答对/生成完成）缺仪式感。本章把 UI 从「像样」推到「有签名」。

### 10.1 动效系统 2.0：从「有动效」到「有签名」

**a) 签名动效「点亮 Light-Up」**（自习室隐喻的动效化，全站唯一记忆点）：
- 首页自习桌进入：一道柔光从左上扫过（模拟开灯），streak 点逐个点亮
- 每日首次学习完成：streak +1 点亮仪式（格子亮起 + 微光扩散 300ms）
- 测验答对：选项绿光轻脉冲；答错：横向 shake 6px
- AI 课生成完成：课程卡「通电点亮」（边缘光扫过一圈）
- 专注入席：屏幕四周暗角渐入（灯光聚焦到桌面的感觉）

**b) 页面转场**：
- 采用 Next 15 实验性 View Transitions（或 framer-motion 模板层 fade+8px 位移），消除页面生硬替换
- 配合每路由 `loading.tsx` 骨架屏（shimmer），转场永远有中间态

**c) 微交互清单**（每个 <200ms，spring 物理感，统一走 motion token）：
| 交互 | 动效 |
|---|---|
| 所有可点元素按压 | scale(0.97)（现有 .studio-press 推广到全站） |
| 投票 | +1 数字向上飞出淡出 |
| 收藏星标 | 星星 spring 弹跳 |
| 复习卡翻面 | 3D flip 400ms |
| 侧栏导航切换 | active 白底块滑动过渡（layoutId），非跳变 |
| Tab 切换 | 指示条滑动 |
| 输入框聚焦 | 边框颜色呼吸过渡 |
| Toast | 底部 spring 弹入 |

**d) 滚动编排克制化**：列表 stagger 间隔 80ms→30ms 且仅首屏可见项；滚动视差全站只允许首页 Hero 一处。

**e) 动效纪律**：所有时长/缓动必须引用 CSS 变量（--dur-*/--ease-*），代码评审禁止内联魔法数字；`prefers-reduced-motion` 全覆盖。

### 10.2 布局系统升级

**a) 自习桌 Bento Grid**：登录首页 dashboard 采用 bento 网格（大小卡错落、栅格严格对齐）——高信息密度 + 秩序感，现代 dashboard 范式。

**b) 卡片三级制**（消灭现在 3-4 种混用风格）：
- `Surface Card`：静态信息（rounded-16, --card 阴影）
- `Raised Card`：可点可悬浮（+ .studio-lift）
- `Feature Card`：深色强调卡（--video-bg 底，每屏 ≤1 张）
全站卡片只允许这三种，圆角/阴影/内边距锁定。

**c) 页面容器阶梯**：内容页 max-w-1080 / 工作台 1160 / 表单流程页 640，区块间距只用 40/56 两档。

**d) 学习工作台布局优化**：视频区 16:9 上限约束、右栏笔记/伴侣面板可收纳、专注模式独立全屏布局（配合 ⑨）。

**e) 平板中间态**：768-1024 目前是缩小版桌面 → 侧栏图标化 72px 专门形态。

### 10.3 视觉打磨清单

- **8pt 间距节奏表**全站对齐：卡内 16/20/24，卡间 16/20，区块间 40/56
- **7 档字级严格执行**（阶梯已定义，执行有漂移，逐页校）
- **图标一致性**：Phosphor 统一 regular（默认）/fill（激活）双态，禁混其他 weight
- **阴影三级制**：--card / --lift / --modal，删除散落自定义阴影
- **深浅双模对比度审计**：ink3/ink4 暗色下逐处 WCAG AA 校验
- **数据可视化语言统一**：热力日历/柱状图/进度条/进度环共用一套红色梯度规则
- 空态插画补暗色适配版

## 11. AI 就绪状态与技术注意（重要）

- **DeepSeek 已充值**，新 key 已写入 `.env`（gitignore，不入库），实测 `deepseek-v4-flash` 生成成功（HTTP 200）。
- **⚠️ v4-flash 是推理模型**：响应含 `reasoning_content`（思维链）。实测 `max_tokens` 太小时会被思维链耗尽导致 `content` 为空。**llm.ts 需适配**：
  1. 各场景 `maxTokens` 需预留思维链空间（建议在原基础上 ×2~3，如造课单节 1500→4000）
  2. 解析时若 `content` 为空但有 `reasoning_content`，视为 tokens 不足，自动放大重试一次
  3. 或对简单场景（搜索扩展/分类）指定关闭推理（若 API 支持 reasoning 开关参数则关闭，降低延迟与成本）
- 全部 AI 场景（造课/导入/伴侣/总结/搜索扩展/回复起草）此前已 mock 验证链路，充值后逐一真跑一遍并按上述适配调参。
