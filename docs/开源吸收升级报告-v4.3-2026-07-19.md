## 二、逐项吸收(5 项,全部落地)

### 1. ts-fsrs → 复习算法换代 FSRS-6(MIT)
- **换了什么**:复习卡调度从简化 SM-2(ease/interval 机械翻倍)升级为 **FSRS-6**(Anki 新一代默认,
  基于 difficulty/stability/retrievability 三变量的 DSR 记忆模型,同等留存下复习次数更少)。
- **落地**:`ReviewCard` 加 8 列(stability/difficulty/state/reps/lapses/elapsedDays/scheduledDays/learningSteps/lastReview,
  migration `fsrs_review_scheduling`);`lib/srs.ts` 保留旧 `scheduleNext`(兼容/测试)+ 新 `scheduleFsrs`;
  PATCH 路由接受**四档 grade(1-4)**并兼容旧两键 remembered;复习页 UI 从「记得/忘了」升到
  **完全忘/吃力/记得/秒答**四键(键盘 1/2/3/4,兼容 ←/→);存量卡 stability/difficulty 为空 → 首评按新卡冷启动。
- **验证**:21 条单测(冷启动/四档单调/成熟卡增长/遗忘转再学习/纯函数);真机点「记得」→ 查库
  `state=1, stability=2.31, difficulty=2.12, reps=1`,与单测输出逐位一致。
- **客户端**:review 页**不引** srs.ts(评分常量内联 1-4 字面量),ts-fsrs 引擎零进 client bundle。

### 2. shiki → 课件代码高亮(MIT)
- **换了什么**:code 块从手写正则高亮升级为 **VS Code 同源 TextMate 着色**(github-light/dark 双主题)。
- **落地**:`courseware-highlight.ts` 异步建 highlighter 单例(20 种语言)+ 同步 `highlightLinesSync`;
  渲染主链 `await ensureHighlighter()` 后,同步渲染里 code 块取 shiki token 生成 **inline color span**;
  未就绪优雅回落手写高亮。**产物是纯静态 HTML(inline style),iframe 内零脚本零外链,天然过 CSP 与安全 lint。**
- **验证**:真机渲染 Python sigmoid 代码——`import/as/def/return` 关键字、函数名、数字、注释、字符串
  全部精确着色,终端镜框+行号保留(截图确认)。现库 89 节暂无 code 块(全英语/口语课),shiki 待新代码课即生效。
- **客户端**:shiki 引擎零进 client bundle(纯服务端渲染期)。

### 3. heti(赫蹏)→ CJK 排版(MIT)
- **换了什么**:中文与半角(英文/数字)之间自动补 ~1/8 em 间距(「学习 English」不再挤成一团),
  + 现代 CSS `text-spacing-trim`/`text-autospace`/`line-break:strict`(全角标点挤压、CJK 严格换行)。
- **落地**:`cjk-spacing.ts` 的 `hetiSpacing`(**标签感知**,跳过 script/style/pre/code,空 span 纯视觉不改文本);
  接入全站共享 `renderMarkdown` + 课件正文;`.hs`/现代排版 CSS 进 globals 与课件 base。
- **验证**:10 条单测(插入/标签感知/幂等/跳过代码);**全库 89 节课件实测 1293 处间距标注**(存量课立即受益)。
- **客户端**:纯 CSS + 服务端产 span,零 JS。

### 4. KaTeX → 公式块(MIT)
- **换了什么**:新增 `formula` 块(第 16 种),数学公式用 **KaTeX 精确渲染**(积分/分式/上下标/求和…)。
- **落地**:`courseware-math.ts` 服务端 `renderToString`(output:html,无 mathml 减体积);
  **CSP 自包含铁律的解法**——剥掉 katex.css 的外链 @font-face,把必要 woff2 **base64 内联为 data: 字体**
  (CSP 已允 font-src data:);**按需注入**:仅含 formula 块的课节才注入 CSS+字体(~180KB base64),无公式课零成本。
- **验证**:真机渲染质能方程/牛顿-莱布尼茨公式/梯度偏导——积分号上下限、分式、希腊字母全部正确显示(截图确认);
  6 条单测(协议校验/自包含/非法 latex 回落纯文本不抛)。
- **客户端**:katex 引擎零进 client bundle;26KB katex 布局 CSS 仅被**服务端 API route** 引用,从不下发浏览器。

### 5. H5P 式交互块 → 填空 + 拖词(借交互设计,自研实现)
- **换了什么**:新增 `fillblank`(填空,键入判分)与 `dragwords`(选词填空,点词→填空/点空→退回)两种交互块。
- **落地**:`courseware-interactive.ts` 确定性渲染 + `INTERACTIVE_RUNTIME` iframe 内判分 JS;
  **移动友好**(不用 HTML5 拖拽,改点选);判分后经 **ct-quiz 协议回传宿主 → 进 LessonQuizResult/错题本闭环**
  (与 quiz 同管道);词库确定性洗牌(块 id 种子)。**只借 H5P 交互范式,不引其 PHP/重运行时。**
- **验证**:真机 JS 驱动实测——填空填对→双绿「全对！」+ ct-quiz{bid,correct:true};
  拖词点选填两空→判分双绿 + ct-quiz 回传;16 条单测(协议校验/段空匹配/渲染纪律/转义/确定性)。
- **客户端**:判分 JS 在沙箱 iframe 内(与课件同源自包含),不进宿主 bundle。

## 六、TTS / 视频:工程决策(不硬塞)

### kokoro-js(浏览器本地 TTS)—— 走已有 E2 注册表,不引 86MB 客户端模型
- **决策**:代码库**已有 E2 多模态注册表**(`models.ts` 的 `modality:"tts"` + `hasModalityAvailable("tts")`,
  与 videoMode 同款),注册一个网关 TTS 模型即自动亮起「跟读音频」能力位,**服务端生成、零客户端重量、
  有确定性兜底**——这正是本项目一贯的架构。
- kokoro-js 浏览器本地方案会:①加 86MB 客户端模型(需自托管,CSP 禁 CDN);②依赖 WebGPU(低端机风险);
  ③中文音质未经验证;④**绕过已有注册表,与架构相悖**。
- **结论**:TTS **走已有注册表**(网关 TTS 模型),kokoro-js 留作「离线/零服务器」场景的**文档化兜底选项**,
  上线前需过中文音质 + 设备 PoC。这是「有更好的模式就用更好的」的负责任吸收决策,而非为落地而落地。

### revideo(程序化视频)—— E1 正式启动时引
- 现有 `export-courseware-video.mts`(playwright 录屏)已是 E1 第一档,零新依赖即可出「课件→分享视频」;
- revideo(MIT,Motion Canvas 服务端渲染 fork)提供模板化转场/BGM 轨,**E1 正式启动、需要模板化预告片时引入**;
- **Remotion 明确排除**(source-available,公司使用 $100/月起,revideo 覆盖同场景)。
