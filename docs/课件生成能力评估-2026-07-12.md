## 一、结论先行（三问三判）

| 问题 | 判断 | 一句话根据 |
|---|---|---|
| (a) 产出真的丰富了吗？ | **部分是** | 文本层真丰富（33/33 起承转合、均 10.7 块/节、9.8 种去重块型、真交互块），但视觉层被撑大——首尾骨架冻结、premium 精修只兑现 3/33 节=9% |
| (b) 真的多样化了（不是换皮）？ | **部分是** | 底层是真结构差异（art 带字体/圆角/纹理/明暗、modeCss 重写版式），但 4 个设计维度里 3 个全语料死钉、热门品类收敛到 ~12 个视觉家族 |
| (c) 真的跟随模型能力提升了吗？ | **否（视觉层甚至反向）** | chat→reasoner：内容持平（+0.3% 字数、深度互有胜负、reasoner 反出一处技术硬伤），视觉层被 45s 硬超时清零（reasoner premium 精修 3→0） |

**总判断**：文本丰富与交互功能是**真的、可直接投用**；但「多样」被高估、「模型跟随」未兑现。好消息是——一切瓶颈的最短修复路径不改模型、不改内容层，只需拆掉管线里三道拦截强模型红利的闸门（见第四节第 1/2/3 条），已经存在的能力即可兑现。

## 二、最有力证据

### (a) 丰富是真的（文本层）
- **块结构**：354 块 / 33 节，均 10.7 块/节，每节去重块型 8–11 种（均 9.8）。concept 仅占 13%、quiz 11.9%，**无任何块霸屏**，不存在「一堆 concept 段落」。
- **起承转合**：33/33 节 `scene/objectives` 开场 → 100% 含 objectives → 100% `summary` 收尾（带 `next` 下集钩子）。
- **内容有真推理链**（非套话）：
  - C 课 premium L4 为「删容器丢数据」的痛点造终端剧情：`docker run --name mydb…` → `docker stop mydb && docker rm mydb` → `# 数据库 test_db 已消失 😱`。
  - E 理财 L3「50/30/20」：月薪 8000 → 到手 6800 → 需要 3400 / 想要 2040 / 储蓄 1360 的完整算例，并**换角色重算**（大二生 1500 元逐项拆）。
  - A 故事课 L3「调膏」：体积比 1:1.5、分两次手法、师徒 6 轮对话。
- **交互块非空壳**（怀疑派实测未能推翻）：quiz 有 `answerIndex`+教学性 `explain` 且解析出现在渲染 HTML、确定性 script 的 `quiz()` 真绑定点击判分；flashcard 真 `front/back` 可翻面；dialog 是真 6 轮 `{speaker,text}`。
- **短板**：`dialog` 仅 7 处（几乎只在 story 课）、`callout` 仅 4 处、5 门课**无一节用 image 块**。

### (b) 多样是真结构差异（非换皮）
- **同题不同模板真分化**：A(story) 每节有 `dialog` 气泡（HTML 类 `dlg/bub/who`）+ 单 quiz + `scroll-lesson` 布局 + 连续主角弧（小林「从学到教」）；B(case) **无 dialog**、改 `callout` + **双 quiz** + `ex--ticket` 工单式例题 + `sidebar-lesson` 布局 + 独立问题案例。
- **跨主题呈现按 art 分化**：Docker→`dev_terminal`（终端窗 chrome、命令行绿 `#3fb950`）、点茶→`storybook/blueprint`、理财→`editorial_paper`（版面横纹 赭 `#b0472b`）。art 每套带**配色+字族栈+圆角(2→24px)+纹理+明暗基底**，`modeCss` 重写宏观版式（终端窗口镜框 / 衬线首字下沉 / 径向节点图）。同输入 C/D 仍分叉出不同 artKey。
- **完全确定性可复现**：视觉链 7 个文件无任何 `Math.random/Date.now`，种子 = `hashSeed(courseId:lessonId:title:sortOrder)`，可单测/回归/截图比对。

### (c) 不跟随模型（甚至反向）— 决定性证据
渲染引擎归因表（C=chat / D=reasoner，同 Docker/classic/premium）：

| 节 | C（chat） | D（reasoner） |
|---|---|---|
| L1 | **llm 成功** 38.8s | deterministic **45.001s 超时** |
| L2 | deterministic（QC 拒：`硬黑投影 rgba(0,0,0,.1+)`) | deterministic **45.000s 超时** |
| L3 | deterministic（同上 QC 拒） | deterministic **45.001s 超时** |
| L4 | **llm 成功** 31.7s | deterministic **45.002s 超时** |
| L5 | **llm 成功** 33.4s | deterministic **45.001s 超时** |
| L6 | deterministic（同上 QC 拒） | deterministic **45.002s 超时** |
| L7 | `course_budget_exhausted` | `course_budget_exhausted` |

- **视觉层反向惩罚**：reasoner 6/6 次 bespoke 全部卡满 45000ms 硬超时被 `controller.abort()` 杀掉；`LlmUsage` 表 `generate_lesson_html` **仅 6 行全属 chat、reasoner 零行**——更强(更慢)的模型从未产出过一版 HTML。premium 精修 3→0。
- **内容层无提升**：C=20069 字 / D=20123 字（+0.3%）；深度互有胜负（D 的 L1 端口映射更细，但 **L4 把 bind mount 误标为「数据卷 Volume」、漏讲命名卷**，是技术硬伤，C 的 L4 反而正确区分卷/绑定挂载）。
- **同质化坐实**：5 门课 designJson `variance/motion/density` **全部 = 7/6/5**；B_case 的 **L4 与 L5 块序列逐字完全相同**。

## 三、结构性瓶颈（具体到文件/机制）

1. **45s 硬超时 + retries:0**（`src/lib/ai/courseware-gen.ts:29,103`）：越强越慢的模型越易撞墙，形成反向淘汰。且**非对称**——块生成走 60s（`src/lib/llm.ts:61`）、bespoke 却更紧的 45s，注释自认「推理模型延迟更高」却明知卡死。**模型不跟随的头号主因。**
2. **bespoke 白名单耦合**（`courseware-gen.ts:165` → `src/lib/ai/models.ts:143-151`）：把生成模型当「显式 bespoke 请求」传入 `selectBespokeModel`，非白名单返回 null 而不回落——premium 用户若没选 opus（默认白名单 4 项里只有 `claude-opus-4-8` 在注册表命中），**交了精修的钱只拿确定性 HTML**。
3. **knobsFor 默认分支**（`src/lib/ai/courseware-design.ts:371-382`）：只对 `silver_english/ai_skill/english_oral` 三赛道特判，culture/tech/finance 全落 default → 恒定 7/6/5，4 个设计维度里 3 个是摆设。
4. **过度规范化裁剪**：`validateBlocks` 只认 13 种块 + `validateCoursewareHtml`/`assessCoursewareDiversity` 规则门（`src/lib/ai/courseware-html.ts:922,968`）把强模型更大胆的 HTML 判违规回落——表现力被规则门裁平。C 的 L2/L3/L6 就是 LLM 真产出了 HTML，却因一条轻量 lint（`硬黑投影`）被**整节拒收**。
5. **质量闭环缺失**：`scoreLesson`/`checkTemplateAdherence`（`src/lib/course-gen.ts:441-468`）发现低质/模板没生效只发埋点，**不重生成、不升模型**——模型再强也不被督促重产。
6. **参数写死 + 输入截断**：maxTokens 全常量（逐节 10000 / 大纲 6000 / bespoke 16000）；bespoke user prompt `slice(0,12000)`、导入素材 `SOURCE_CTX_MAX=6000`——强模型的长上下文/长输出能力用不上。

## 四、下一步升级建议（按投入产出比排序）

| # | 改动 | 位置 | 预期收益 | 投入 |
|---|---|---|---|---|
| **1** | **动态化 bespoke 超时 + 加重试**：按模型延迟档设超时（reasoner/opus 给 90–120s），retries≥1 | `courseware-gen.ts:29,103` | **收益最大**——救活当前 91% 走不通的 premium 精修路径，是「模型跟随」能否兑现的总开关 | 低 |
| **2** | **修 bespoke 白名单耦合**：让 premium 恒用白名单首个可用强模型精修，而非锁死在生成模型（传 `undefined`） | `courseware-gen.ts:165` | **一行改动**让 premium 名副实，不再「交钱拿确定性」 | 极低 |
| **3** | **QC 软违规自动修正而非整节拒收**：`硬黑投影` 这类样式违规做后处理归一化，别丢整节已产出的 HTML | lint 门（`courseware-html.ts`） | 低成本挽回 chat 现在被白扔的 3/7 节 | 低 |
| **4** | **解冻数值三维**：给 culture/tech/finance 也配差异化 variance/motion/density 预设 | `courseware-design.ts:371-382` | 廉价扩多样，把「一维 artKey 选择」还原成真多维 | 低 |
| **5** | **给模板遵循度装重生成闭环**：`checkTemplateAdherence` 不达标（story 缺 dialog、case 节间复制）触发一次重生成或换更强模型 | `course-gen.ts` | 同时提丰富度与模型跟随，把「只观测」变「闭环」 | 中 |
| **6** | **放开单节 maxTokens 随模型放大 + 解耦 mode 与 art**（让同 art 可配多 mode） | 写死常量 + `courseware-catalog.ts` ART_TO_MODE | 让更强模型能写更深、同一视觉家族内拉开版式 | 高（长期项） |

**执行建议**：先做 **1+2+3**（都不改模型、不改内容层，只拆三道拦截闸门），成本一两天，即可让「模型跟随」从「否/反向」翻转为「正向」、premium 精修覆盖率从 9% 拉到接近满覆盖。4/5 作为多样性与质量闭环的中期项，6 作为长期架构项。

## 附：与历史「六模板同质化」的关系

历史顽疾（只换色 / 六模板套一骨架 / 每节雷同）**主体已消除**——art 现在带字体+形态+明暗（不只 accent）、modeCss 重写版式、variance 打散块布局，跨 art 的课确实判然不同。但残留一个**缩小版残影**：内容 hint + 赛道候选把热门品类漏斗到少数 art，同类课件收敛为 **~12 个视觉家族、家族内互为「同世界换家具」**；叠加 designJson 数值三维死钉、case 模板节间复制。这不是回退，而是「规整被误读成丰富」，靠上表第 4/6 条继续拉开。
