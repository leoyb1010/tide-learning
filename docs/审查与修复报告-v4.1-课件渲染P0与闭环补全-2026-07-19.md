## 二、本轮发现与修复(按严重度)

### P0 · 精品课件脚本被父页 CSP 整体拦截 —— 主力功能在 dev 与生产全废
- **根因**:7-13 审计提交(6de24d5/89393a8)给 middleware 加了 `script-src 'self' 'nonce-…'` 的严格 CSP;
  **srcdoc iframe 会继承父文档 CSP**,课件内联运行时无 nonce → 脚本被拦。表现为:课件只剩 CSS 装饰、
  文字全部停在 `[data-reveal]{opacity:0}`、无翻页器、无 ct-ready/ct-height/ct-quiz——**v4 的 D1/D2 闭环在真机上全部是死的**。
- **为什么三轮审计都没发现**:课件快照/渲染检查脚本(courseware-snapshots/render-check)全部**独立渲染**课件 HTML,
  绕过了 App 内嵌路径;单元测试不跑浏览器。快照绿≠产品好——这是本轮最大的流程教训(改进项见 §五.3)。
- **修复**:服务端页面读取 middleware 的 `x-nonce`,经 Player 传入 HtmlCourseware,渲染前把 nonce 注入课件
  HTML 的 `<script>` 标签(`injectNonce`,只对已过安全 lint 的我方管线产物注入,sandbox="allow-scripts" 铁律不变)。
  改动:`HtmlCourseware.tsx`(injectNonce + nonce prop)、`Player.tsx`(cspNonce 透传)、`learn/[lessonId]/page.tsx`、`preview/page.tsx`。
- **验证**:预览页与学习页课件文字/翻页器/分步揭示/完课卡全部实测正常(见 §一)。

### P1-1 · 换课后课件答题静默丢报(quiz 去重集跨课残留)
- `HtmlCourseware` 的 `reportedQuizRef`(Set)从不按课清空;App Router 软导航下组件实例复用(learn 页 `<Player>` 无 key),
  而 v4 起新课块 id 是确定性 `blk_0…`,第二节起同 bid 的答题上报被 `has(bid)` 拦截。
- **修复**:`Player` 以 `key={lesson.id}` 挂载 HtmlCourseware——换课强制重挂,顺带清掉 pagedReady/page/height 跨课残留 state。

### P1-2 · HTML 课件零进度上报——D1 激励接线对主力课型无效
- 全库 89 个 ai_block 节全部有 htmlJson → 走 HtmlCourseware 分支;该分支收 `ct-page` 只更新页码 UI,从不 POST /api/progress。
  v4 自称「学习进度接线激励系统」,但对占比最大的课型 streak/完课/潮汐日历依旧恒空(答题/笔记除外)。
- **修复**:HtmlCourseware 增加 `onPage` 回调;Player 接到现成的 `reportBlockPage`(页序进度+末页完课)与 `onBlockComplete`(下一节卡)。
- **验证**:实测翻页后 `lastSlideIndex=2→10`、`completedAt` 落库、完课卡弹出、`StreakDay` 当日 minutes 增加。

### P2-1 · 集市过审后改文案不复审(内容审核 TOCTOU)
- `market/share` 语义1(action=update)与语义2(已上架经营)改 title/subtitle **不过任何审核**(代码注释自认),
  干净文案过审后可改成违规/引流文案白嫖集市展示位。

  引导走语义3 重新上架送审(那条路 review 级会正确落 pending 人工复核)。语义3 行为不变。

### P2-2 · 「免登录试读」按钮对 8/23 门已发布课是 404 死链

## 三、遗留技术债与风险(未修,按优先级入下轮)

1. **B5 复用是颗上膛的雷(未来 P0)**:`renderAndStoreLessonHtml` 的 enhance 复用路径只看 `renderEngine==="llm"`,
   不校验 blocks 是否变化;一旦未来补上任何「重生成」入口(gen-quality 看板的重生成按钮是蓝图 C1 的明确方向),
   旧 blocks 的 bespoke HTML 会被盖上新 sourceHash 永久端给用户。本轮已加写侧保险丝(§二.小项3),
   但**任何新增的 blocksJson 写点都必须同事务清 html 三字段**——建议直接抽一个 `rewriteLessonBlocks()` 唯一写入口。
2. **regen 后集市不复审**:对已上架课重生成课节(未来入口)不会复位 sharedStatus。与 §二.P2-1 同族,
   规则建议:`sharedStatus ∈ {shared,pending}` 的课重生成任何节 → 复位 pending。
3. **审计盲区:快照≠产品**。课件管线的所有机检(render-check/snapshots)都在 App 外渲染,CSP/宿主握手/进度上报这类
   「嵌入层」问题全部检不到。蓝图 C3(渲染烟囱)应升级为:**headless 打开真实 learn 页**断言 ct-ready 收到 + 首屏文字可见,
   进 CI(playwright 已在 devDependencies)。这条能直接防住本轮 P0 的整个 class。
4. **课件无续读**:HTML 课件每次进入都从第 1 页开始(`initialSlidePage` 只给了 BlockSlideshow)。
   协议已有 `ct-nav`,补一条 `ct-goto{page}` + 挂载时由宿主发送即可,半天工作量,体验收益大。
5. **article/mixed 课无完课路径**(存量):不渲染播放器则 `time` 永不推进,没有任何进度可写。
   建议 article 类加「标记读完」按钮或按滚动到底完课(对齐块课滚动模式语义)。
6. **soft-404**(存量,全站):流式 SSR 下 notFound() 返回 HTTP 200 + 404 UI,SEO 弱化。已知取舍,
   如要修:入口页在流式开始前先做存在性查询(或中间件级校验)。
7. **OG 图依赖 Google Fonts 运行时抓取**:内网/无外网环境字体子集抓不到会回落无 CJK 字体(豆腐块风险),
   建议本地内置 Noto Sans SC 子集文件兜底。
8. **repo 卫生**:report*/ 五个目录共 ~160MB 未跟踪截图与 15 个一次性 capture/verify 脚本,建议归档清理或 gitignore;
   `logs/next-start-3100.log` 已在 gitignore 范围。
9. **调参观察**:gpt-5.5 标 fast(45s)却配 32k maxOutput,bespoke 慢产出场景偏紧;C1 纠偏 regen 未传 timeoutMs(默认 60s),
   opus/slow 档偶发超时回退第一版(已兜底,只损效率)。跑一轮真实生成矩阵后按数据调。
10. **S1 版本化只有 rerender 存档**:`LessonRevision.reason` 宣称的 regen/manual 与「blocksJson 真值存档」尚无写入方;
    落 §三.1 的 `rewriteLessonBlocks()` 时一并补齐。

**已审查并排除**(有代码证据,不再复查):quiz-result IDOR/越权写、复习卡任意内容刷库、免费积分月界/订阅翻转重领、
删课重造刷配额、免费用户触发 premium bespoke、preview 泄露付费/私有/草稿内容、postMessage 未校验 source(实际有校验)、
OG SSRF、gen-quality 越权、progress 单位错配、C1 计费权重(按实际模型 costWeight 扣,无漏账)、schema/migration 漂移。
