# GitHub 吸收进化候选清单 · 2026-07-19

> 目标:为潮汐学习(AI 造课 + 自包含 HTML 课件 + SRS 复习 + 集市)找「能吸收的开源项目」。
> 筛选铁律:①license 可商用(我们是公司主体,source-available 需付费的标红);②尊重课件 CSP 自包含约束
> (**任何东西都不进 iframe 运行时**,只允许「服务端产 HTML/CSS」或「宿主页运行」);③吸收方式分三档:
> **A=直接引库**(算法/构建期工具)、**B=抄思想/抄规范**(不引代码)、**C=远期储备**。

## 二、每项的落法(工程视角)

### 1. ts-fsrs —— 复习算法换代(最干净的一次吸收)
- 现状:`ReviewCard` 用 intervalDays/ease 的简化 SM-2,打分只有 remembered 布尔。
- 落法:服务端引 `ts-fsrs`(纯 TS 零依赖);Card 表加 stability/difficulty/state/lastReview 四列(migration);
  打分从二值改四档(Again/Hard/Good/Easy,UI 加两个按钮);存量卡首评时冷启动迁移(FSRS 对无历史卡有默认参数)。
- 关联资产:错题转卡(quiz-result)、复习室、到期计数全部无缝受益。
- 参考:[ts-fsrs/fsrs.js](https://github.com/open-spaced-repetition/fsrs.js)、[awesome-fsrs](https://github.com/open-spaced-repetition/awesome-fsrs)、[FSRS 算法说明](https://github.com/open-spaced-repetition/fsrs4anki/wiki/The-Algorithm)。

### 2. shiki —— code 块高亮(构建期,不进运行时)
- `codeToHtml(code, {lang, theme})` 在 **renderCoursewareHtml 阶段**执行,输出带 inline style 的 HTML,
  iframe 内零 JS、零外链,天然过我们的安全 lint。
- 主题按 art 双份(亮/暗)或用 css-variables 主题对齐 `--ct-*`;bundle 只装需要的语言(python/js/ts/sql/bash…)。
- 全库 dev_terminal 28 节直接受益;重渲一次生效。

### 3. heti —— 中文排版规则(抄 CSS,不引 JS)
- 吸收:标点挤压(`heti--punc`)、中西文间距、行首行尾禁则相关的纯 CSS 子集,并入课件 base CSS 与全站 prose 样式;
  其 JS 部分(自动加 span)不要——我们内容是服务端产 HTML,可在渲染器里做同等标注。

### 4. KaTeX —— 公式块(蓝图 B5 的正解)
- 新块 `formula`(latex 字段)→ 服务端 renderToString → 内联 HTML;KaTeX 字体子集化后 base64 进 CSS。
- 注意体积:每节只在含 formula 块时注入字体;或全站公共 woff2 由宿主页加载、课件内用降级栈(需实测)。

### 5. H5P —— 交互块菜单(只抄设计)
- 优先吸收三种:**填空**(cloze,判分回传走现有 ct-quiz 协议扩展)、**拖词入句**(drag-the-words)、
  **热点图**(image hotspot,可与 diagram 块组合)。
- 每种 = blocks.ts 新类型 + 确定性渲染 + runtime 判分 + LessonQuizResult 复用,D2 闭环自动承接。

### 6. kokoro-js —— 浏览器本地 TTS PoC
- PoC:宿主页(Player)加「朗读本页」按钮,kokoro-js WebGPU 生成、宿主播放;课件经 ct-* 协议高亮当前句。
- 决策点:中文 v1.1 音质是否达标(银发用户对音质敏感);不达标则等 NewAPI 网关上 TTS 模型(E2 注册表加一行即接入)。
- 参考:[kokoro-js 发布](https://huggingface.co/posts/Xenova/620657830533509)、[Kokoro-82M-ONNX](https://huggingface.co/onnx-community/Kokoro-82M-v1.0-ONNX)、[kokoro-onnx](https://github.com/thewh1teagle/kokoro-onnx)。

### 7. revideo —— E1 视频管线(启动时再引)
- 当前 export-courseware-video.mts(playwright 录屏)已能出片,零新依赖——E1 第一档维持现状;
- 需要「模板化预告片/转场/配乐轨」时引 revideo(MIT、服务端渲染 API、Motion Canvas 语法);
- [Remotion](https://github.com/remotion-dev/remotion) 明确排除:source-available,[公司license $100/月起](https://github.com/remotion-dev/remotion/blob/main/LICENSE.md)([对比资料](https://www.pkgpulse.com/guides/remotion-vs-motion-canvas-vs-revideo-programmatic-video-2026))。

## 三、明确「不吸收」及理由
- **Remotion**:license 商用付费,revideo 覆盖同场景。
- **mermaid/markmap 运行时**:需 iframe 内 JS + 布局引擎,破 CSP 自包含;其「图形语言」已被我们的 diagram 块以确定性方式吸收,后续扩 kind(tree/matrix)继续走自研。
- **H5P 运行时/框架**:PHP+重运行时,只取内容类型设计。
- **edge-tts 进生产**:非官方接口 + GPL,商业风险;仅原型对照。
- **同类 AI course generator 整体**:架构落后于现状,无整体吸收价值。

## 四、建议的吸收顺序(与现有蓝图合流)
1. **ts-fsrs + shiki**(各半天~一天,独立可并行,立收):复习科学性 + 代码课质感。
2. **heti CSS 子集**(半天):全站+课件 CJK 排版细则。
3. **KaTeX formula 块**(1-2 天,含字体子集实验):理科课解锁,接进 15+1 块协议与 QC。
4. **H5P 式交互块**(每种 1 天,按需):填空→拖词→热点。
5. **kokoro-js PoC**(半天定生死)→ 达标则进 E2;否则等网关模型。
6. **revideo**:E1 正式启动时引。
