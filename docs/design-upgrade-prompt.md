# 有道自习室 STUDIO · 视觉/动效/UI/布局 全面升级作战 Prompt

> 用途：一次性、彻底地把 Web 端（Next.js）+ iOS 端（SwiftUI）的界面从「工程能用水平」拉到「商业化上线 100 分水平」。
> 这份文档就是给执行方（你或 AI）的**完整任务指令**——照它排查、照它升级，不需要再补充。
> 基线：Web v2.3（顶部导航壳 + STUDIO token）｜ iOS v1.0（17 模块，SwiftUI）。

---

## 0. 为什么现在「不够看」——先认清病灶（照此逐项验伤）

当前两端都停在「有设计系统，但把系统用得单调」的阶段。具体症状：

| 病灶 | 现状 | 商业化标准 |
|---|---|---|
| **表面太平** | 所有卡片都是 `surface 白底 + border + 一层软阴影`，全站一个质感 | 分层材质：主卡/次卡/浮层各有不同的底、边、光、投影层级 |
| **色彩太素** | 中性灰蓝 + 7% 红，几乎不用第二强调色、不用渐变、不用氛围色 | 克制但有层次：品牌红 + 1-2 个功能辅助色 + 场景化渐变 + 氛围光晕 |
| **留白太挤或太散** | 间距靠零散 padding，无统一节奏；信息密度忽高忽低 | 8pt/4pt 网格贯穿，section 呼吸感统一，视觉韵律稳定 |
| **动效太浅** | 只有进场 rise + hover lift + 按压 scale，几乎无编排、无转场、无微交互反馈 | 有进场编排（stagger）、路由转场、数据变化动画、手势反馈、加载态骨架的高级感 |
| **排版太单** | 字号层级少（大标题/正文/辅助三档），无 eyebrow 节奏、无引导段、无强调排版 | 5-6 档层级 + 数字用等宽 + 关键信息有排版重音 + 中英混排讲究 |
| **元素太孤** | 图标/徽章/标签都是标准款，无品牌插画、无自定义图形、无空态艺术 | 品牌图形语言：自定义空态插画、成就徽章体系、数据可视化、有记忆点的图形 |
| **首屏太弱** | 营销首页信息堆叠，无叙事节奏、无 hero 冲击力、无 scroll 惊喜 | 首屏有 hook，中段有「第二眼惊喜」，滚动有节奏，转化路径清晰 |
| **一致性漏** | 部分页面还用旧 token、iOS 部分屏纯色块无质感 | 全站/全端 token 零漏网，每个屏幕都达线 |

**核心判断标准**：把任意一屏截图，放到 Linear / Notion / Vercel / Cron / Arc / Raycast 这些「设计标杆产品」旁边对比——如果一眼看出是「AI 生成的通用界面」，就是不及格。目标是让每一屏都经得起这个对比。

## 1. 统一设计语言纲领（两端共同遵守，先立后改）

### 1.1 设计哲学一句话
**「安静的自习室 + 有温度的仪式感」**：整体克制、专注、留白充足（自习室的安静），但在「学习被记录、进步被看见、目标被点亮」的关键时刻给强反馈（仪式感）。红是唯一的高饱和信号色，只在「此刻重要」的地方出现。

### 1.2 色彩系统升级（从 2 层扩到 4 层）
- **中性基座**（冷灰蓝，保留）：`bg / surface / surface2 / surface-inset / ink1-4 / border1-2`。**新增要求**：每个表面之间要有可感知的层次差（不是都白），用 1-2% 的明度/色相微差拉开卡片、浮层、内嵌区。
- **品牌红**（保留，`#fc011a`）：信号色，全站 ≤7% 面积。**升级**：定义 red 的完整状态梯度（default/hover/active/soft/soft-border/glow），glow 用于关键 CTA 的柔光。
- **功能辅助色**（新增，低饱和）：成功绿 / 警示琥珀 / 信息蓝——只用于状态语义（完课✓、待复习、通知点），不参与装饰。各定义亮暗双值。
- **场景化渐变 + 氛围光**（新增，关键升级）：
  - 深色区（学生证/AI 卡/沉浸学习页）：对角渐变 + 单点柔光晕（radial glow），不是死黑平面。
  - 课程封面：按赛道映射不同渐变色系（已有雏形，需系统化成「赛道色板」）。
  - hero / 空态：极淡的品牌色氛围光作背景层次。

### 1.3 材质与层级（Materiality，最缺的一环）
定义 4 级「表面海拔」，每级有专属的 底色/边框/投影/内高光：
- **L0 页面底**：`bg`，无阴影。
- **L1 主卡**：`surface` + `border` + 软投影 + 极淡内顶高光。
- **L2 浮起卡/hover 态**：抬升投影 + 更亮边缘。
- **L3 浮层**（下拉/弹窗/Sheet）：`lift` 强投影 + 毛玻璃背景 + 边缘高光。
- 深色「Color Block」是独立材质：渐变 + 柔光 + 细纹理，每屏 ≤1 处，用于最高优先级的展示（学生证/hero/AI 建议）。

### 1.4 排版系统（扩到 6 档 + 讲究）
- 层级：`display(hero大标题) / h1 / h2 / h3 / body / caption`，字号与字重成系统（不是随手调）。
- **数字/学号/时长/积分/百分比** 一律等宽字体（IBM Plex Mono / iOS SF Mono），这是「专业感」的关键细节。
- **eyebrow**（小型大写追踪标签）：按「每 3 个 section 最多 1 个」配额使用，不滥用。
- **中英混排**：中文正文行高 1.6-1.75，英文/数字字距微调；标题可中英同现但层级分明。
- **强调排版**：关键句可用「更大字号 + 更重字重 + 品牌红一个词」制造视觉重音。

### 1.5 间距与栅格（8pt 系统贯穿）
- 基础单位 4pt，主用 8/12/16/24/32/48/64。所有 padding/gap/margin 从这套取值。
- section 之间的呼吸：桌面 64-96px，移动 40-56px，节奏统一。
- 内容宽度阶梯：窄（阅读 60ch）/ 中 / 宽（仪表盘），不是所有页都一个 max-width。

### 1.6 图形语言（品牌记忆点，需从零补）
- **空态插画**：每类空态（无笔记/无课程/无复习/无通知/无网络）一张有品牌感的极简插画或图形，不是一个灰图标。
- **成就徽章体系**：徽章要成套设计（形状/配色/等级质感一致），不是随机 emoji/图标。
- **数据可视化**：学习节奏、复习曲线、连续天数、积分消耗——用有设计感的图表（不是默认柱图），配色进 token。
- **自定义图标节奏**：主导航/关键动作用一致的图标族（Phosphor / SF Symbols 已选定），weight 与 fill 状态成系统。

### 1.7 动效系统（Motion，最能拉开差距的一环）
定义 4 类动效，每类有明确时长/缓动：
- **进场编排**（stagger）：列表/卡片入场按序 60-80ms 递延，不是齐刷刷出现。缓动用 ease-out-expo。
- **状态转场**：路由切换、Tab 切换、视图切换有转场（淡入+位移 or 共享元素），不是硬切。
- **数据动画**：数字滚动（积分/票数/进度）、进度条弹性增长、连击跳动、复习卡飞出、点亮/通电——关键数据变化都有动画。
- **微交互反馈**：按压 scale、hover 抬升+高光扫过、磁吸（主 CTA）、下拉回弹、开关弹性、涟漪。
- **签名动效**（品牌记忆点）：「点亮自习室」的 lightUp/sweep、复习卡 3D 翻牌、学生证入场——这几个要做到惊艳，是产品的「设计钩子」。
- **铁律**：全部尊重「减弱动态效果」（`prefers-reduced-motion` / `UIAccessibility.isReduceMotionEnabled`）降级为即时切换；动效只用合成友好属性（transform/opacity），不做重排。

### 1.8 双主题一致性
- 亮/暗跟随系统 + 手动覆盖。每个新增的色/材质/渐变/光晕都必须定义亮暗双值，暗色不是简单反色（要重新调对比度与氛围）。
- 深色区在亮色主题下是「Color Block」，在暗色主题下要与页面底融合但仍有层次。

## 2. Web 端（Next.js / Tailwind v4 / framer-motion）升级任务

### 2.1 排查（先全量验伤，输出问题清单再动手）
1. **token 漏网扫描**：grep 全站 `accent-600/accent-700/ink-100/paper-raised` 等旧 token 残留，以及任何硬编码色值（`#`开头、`rgb(`、`rgba(`），列出每一处，全部迁移到 STUDIO 语义 token。
2. **材质单一扫描**：找出所有 `bg-[var(--surface)] + border + shadow` 的千篇一律卡片，按 §1.3 的 L0-L3 海拔重新分级。
3. **动效缺失扫描**：列出所有列表/网格（该有 stagger 却齐刷）、所有路由切换（该有转场却硬切）、所有关键数字（该滚动却静态）。
4. **排版扁平扫描**：找出层级不足、eyebrow 滥用、数字没用 mono 的地方。
5. **空态/加载态扫描**：找出所有用灰图标当空态、用简单 spinner 当加载的地方。
6. **响应式断点扫描**：桌面/平板/移动三档，找出布局挤压、字号不适、触达区过小。

### 2.2 逐页升级（每页都要达线，按优先级）
> 每页交付标准：截图能与标杆产品同框不露怯。

- **营销首页 `/`**（转化门面，最高优先级）：hero 要有冲击力（大标题排版重音 + 氛围光 + 产品预览动效）；三引擎/学习闭环用有节奏的 bento 或分栏，不是等宽卡堆；滚动有进场编排；每屏有「第二眼惊喜」（一个精致的小交互/图形）。
- **书桌 `/desk`**（登录首页）：问候区有仪式感（今日状态点亮动效）；学习中卡有真实缩略质感；三卡材质分级；AI 建议深色卡用渐变+柔光；氛围条 live 呼吸动效。
- **学习台 `/courses/[id]/learn/[lessonId]`**（核心体验）：沉浸感（可选深色模式）；视频/块课件/图文三态都要精致；边学边记的 Sheet 交互流畅；进度条水位质感；专注模式仪式感。
- **课程库/详情**：封面渐变按赛道系统化；卡 hover 抬升+高光；详情页大纲有层次；订阅门用有说服力的 paywall（不是干巴巴一句话）。
- **笔记馆/详情/笔记本**：五视图切换有转场；笔记卡有来源质感；AI 整理结果展示精致；采集面板四入口有设计感。
- **复习室/考试**：复习是「签名动效」重点——任务卡入场、卡堆展开、3D 翻牌、飞出、连击、结算 confetti 全部做到惊艳；考试成绩单有仪式感。
- **成长档案/学生证**：学生证是「设计钩子」——纸质证件质感、大数字排版、等级称号、真二维码、格言、Swift Charts 级别的数据图；成长足迹有徽章体系与热力日历。
- **社区/广场/集市**：Feed 流卡有微博/X 级的信息密度与互动质感；投票有「本周之星」大卡与趋势动效；集市课卡有分享感。
- **设置/订阅/积分**：分区清晰有层次；积分卡有余额动效与消耗曲线；订阅套餐卡有档位对比说服力。
- **通用**：顶部导航（毛玻璃+active 滑动指示）、下拉/弹窗（浮层材质+进场）、Toast、空态插画、骨架屏、404/error 页——每个都要达线。

### 2.3 Web 交付验收
- 无任何硬编码色/旧 token（grep 为 0）。
- 每类动效都在，且 `prefers-reduced-motion` 完整降级。
- 亮暗双主题每页都对（暗色不是反色）。
- 三档响应式无挤压/溢出。
- `tsc 0 错 · build 成功`，生产模式各页 SSR 保持 <300ms（不因动效拖慢）。
- 关键页截图与标杆产品同框自评 ≥90 分。

## 3. iOS 端（SwiftUI）升级任务

### 3.1 排查（对齐 Web 但用原生手法）
1. **纯色块扫描**：找出所有 SwiftUI 里 `.background(Studio.surface)` 单层卡片，按海拔分级（用 material/shadow/gradient）。
2. **系统默认扫描**：找出用系统默认过渡、默认列表样式、默认 spinner、灰 SF Symbol 当空态的地方。
3. **DTO→UI 完整度**：确认每个屏幕的数据都完整展示（不因字段缺失留空白）。

### 3.2 原生化升级（用 SwiftUI/iOS 的长处）
- **材质**：用 `.ultraThinMaterial`/`.regularMaterial` 做浮层毛玻璃；卡片阴影分级；深色区用 `LinearGradient` + `.blur` 柔光晕。
- **动效**：
  - 列表进场 stagger（`.transition` + 递延 `.animation`）。
  - 视图切换用 `matchedGeometryEffect`（共享元素转场，比如课卡→详情、复习卡翻面）。
  - 数字滚动用 `.contentTransition(.numericText())`（iOS 原生数字动画）。
  - 复习卡 3D 翻牌用 `.rotation3DEffect`，飞出用 `.offset`+旋转+淡出。
  - 下拉刷新、手势评分（左右滑）、按压反馈（scale）。
  - 全部尊重 `@Environment(\.accessibilityReduceMotion)`。
- **图表**：学习节奏/复习曲线用 **Swift Charts**（已在成长档案用了，推广到所有数据展示）。
- **触觉反馈**：关键操作（点亮/完课/评分/连击/发布）加 `UIImpactFeedbackGenerator`——这是移动端「高级感」的独门武器，Web 做不到。
- **字体**：打进 Plus Jakarta / Noto Sans SC / IBM Plex Mono（当前用系统字体，与 Web 不完全一致）；数字用 SF Mono/Plex Mono。
- **空态/骨架**：每类空态一张品牌图形（可复用 Web 的 SVG 思路），骨架用 `.redacted` + shimmer。
- **深色模式**：Asset Color Set 亮暗双值，每屏走查。
- **TabBar/导航**：自定义 TabBar 质感（当前用系统 TabView，可保留但要调 tint/材质）；导航栏大标题排版。

### 3.3 逐屏升级（对齐 Web 的逐页清单，17 屏全覆盖）
认证/书桌/课程库/详情/学习台/造课/笔记馆/详情/笔记本/复习室/考试/成长档案/积分/订阅/设置/社区/集市/个人主页/通知/伴侣——每屏达线，重点打磨「书桌、学习台、复习室、成长档案、造课过程剧场」这 5 个体验高峰。

### 3.4 iOS 交付验收
- `xcodebuild BUILD SUCCEEDED`。
- 模拟器逐屏实测：每屏真实数据渲染 + 无纯色块单调感。
- 亮暗双主题走查。
- reduce motion 降级正确。
- 关键屏截图与 App Store 精品应用（如多邻国/Notion/Things/Fantastical）同框自评 ≥90 分。

## 4. 执行方式建议（怎么把这份 prompt 变成行动）

### 4.1 推荐流程（避免"改了这里乱了那里"）
1. **先立系统**：先把 §1 的设计语言落成两端的 token/组件层升级（色板扩展、材质分级、动效工具类、排版规范、图形资产）。这是地基，一次立好。
2. **再改页面**：基于新地基逐页/逐屏升级，每改完一页就截图自评（对标杆）。
3. **验伤在前**：每一步先输出「问题清单」（§2.1 / §3.1 的排查），再动手，避免遗漏。
4. **对抗验收**：改完做一轮「全盘设计 review」——多视角审查（材质/色彩/动效/排版/一致性/无障碍各一个维度），把不达线的挑出来重改。
5. **双端对齐**：Web 和 iOS 的同一功能屏，视觉语言要一致（同样的材质哲学、同样的动效性格、同样的排版层级），但各用平台长处（iOS 有触觉/共享元素转场，Web 有 CSS 动画/毛玻璃）。

### 4.2 参照标杆（做的时候放旁边对比）
- **克制高级**：Linear、Vercel、Cron/Notion Calendar、Arc。
- **学习产品**：Duolingo（仪式感/动效/图形语言）、Notion（信息层次）、Readwise（笔记/复习体验）。
- **移动精品**：Things 3、Fantastical、Bear（排版）、Duolingo（触觉/动效）。

### 4.3 什么算「100 分商业化水平」（验收心法）
- 任意截图放到标杆产品旁，看不出是"AI 通用界面"。
- 每屏有「设计意图」——为什么这样排、为什么这个色、为什么这个动效，都能说清。
- 有「记忆点」——用户会记住学生证、会记住复习翻牌、会记住点亮自习室的那一下。
- 细节经得起放大——间距对齐、字距、数字等宽、边缘高光、投影柔和度、动效缓动。
- 无障碍达标——对比度 AA、reduce motion、动态字体、VoiceOver（iOS）。
